About ClinNoteRAG

An agentic retrieval-augmented system that automatically evaluates medical student patient history notes against NBME clinical rubric concepts — with zero labeled training data.

What it does

When a medical student writes a patient history note during a clinical skills exam, they need to document specific clinical concepts — chief complaint, relevant history, associated symptoms, and social/family history. ClinNoteRAG reads the free-text note and checks it against a rubric of required concepts, reporting which were documented and which were missed.

Unlike traditional automated scoring systems that require hundreds of labeled training examples, ClinNoteRAG uses large language model reasoning to evaluate notes with no prior training on the target case. This means it can be applied immediately to new cases as they are added to the exam pool.

System architecture

ClinNoteRAG is built on an Agentic RAG architecture — a large language model that actively queries a vector database as a tool, reasoning over each clinical concept individually rather than processing everything in a single pass.

Agentic RAG Pipeline

Production architecture

For every submitted note, the agent works through each required clinical concept one at a time. It retrieves semantically similar synonyms and phrasings from ChromaDB, then reasons about whether that concept is documented in the note — producing a present/absent verdict with supporting evidence.

1
Concept retrieval The agent calls ChromaDB with each concept as a query, fetching the closest synonym variants stored from the NBME rubric.
2
LLM reasoning The language model receives the note text alongside the retrieved synonyms and determines whether the concept is present, absent, or uncertain.
3
Evidence extraction When a concept is found, the agent extracts the exact phrase from the note that supports the verdict — shown highlighted in the result.
4
Score aggregation Verdicts across all concepts are compiled into a final coverage score, with each concept marked present or missing.

Clinical cases covered

The system is evaluated on 10 NBME USMLE Step 2 standardized patient cases provided by NBME. Each case has 13–18 required clinical concepts.
